# Project Context: Speechbrain Dashboard (Frontend)

## Overview
The Speechbrain Dashboard is a modern web application built to serve as the user interface for the Speechbrain audio processing backend. It allows users to authenticate, upload audio files, and view rich transcriptions, summaries, and analytics derived from their audio content. 

## Tech Stack
- **Framework:** Next.js (version 15+ using App Router), React 19.
- **Styling:** Tailwind CSS v4, `@tailwindcss/typography`, `clsx`, `tailwind-merge`.
- **Animations:** Framer Motion for dynamic micro-animations.
- **Icons:** `lucide-react`.
- **Authentication & Database:** Firebase Client SDK (`firebase` v12).
- **Data Visualization & Rendering:** 
  - `recharts` for analytics/charts.
  - `react-markdown` for rendering formatted summaries and NLP outputs.
- **Language & Tooling:** TypeScript (`tsconfig.json`), ESLint (`eslint.config.mjs`).

## Architecture & Folder Structure
The application follows standard Next.js App Router conventions with a clear separation of UI components, state management (Context), and external integrations (Firebase).

### 📁 Root Directory
- **`package.json`**: Lists Next.js, React, Tailwind, Firebase, and UI dependencies.
- **`next.config.ts`**: Next.js configuration.
- **`tailwind.config` / CSS Configs**: Contains Tailwind 4/PostCSS configs for design system variables.
- **`.env.local`**: Environment variables (e.g., Firebase client credentials).

### 📁 `src/app/`
Contains the Next.js App Router pages and layouts.
- **`layout.tsx` / `page.tsx`**: Root layout and landing page of the application.
- **`providers.tsx`**: Wraps the application with necessary context providers (e.g., AuthProvider).
- **`globals.css`**: Global stylesheet including Tailwind directives and potentially custom CSS variables.
- **📁 `login/`**: 
  - Authentication views for users to sign in or register.
- **📁 `dashboard/`**: 
  - The main authenticated interface where users view and manage their transcribed sessions, insights, and settings.

### 📁 `src/context/`
Manages global React state.
- **`AuthContext.tsx`**: Connects to Firebase Authentication. Exposes user state and login/logout functions to the rest of the app, ensuring protected routes (like `/dashboard`) remain secure.

### 📁 `src/lib/`
Utility modules and service initializations.
- **`firebase.ts`**: Initializes the Firebase Client app with credentials from `.env.local`.
- **`firestore.ts`**: Contains helper functions and configurations to query or mutate data in Firestore (e.g., fetching a user's transcription history).

## Design & Workflow (Context for LLMs)
- **Auth Flow**: Users land on the app, navigate to `/login`, and authenticate via `AuthContext.tsx` using Firebase.
- **Data Fetching**: Authenticated users in the `/dashboard` route fetch their processed audio sessions from Firestore using functions in `src/lib/firestore.ts`.
- **UI UX**: The UI heavily utilizes Tailwind CSS for responsive design and `framer-motion` for smooth transitions. Summaries generated by the backend (mT5/Gemma) are rendered safely using `react-markdown`, and analytics are plotted via `recharts`.
